# 树莓派家庭监控摄像头产品需求文档（PRD）

- **文档版本**：V0.2
- **状态**：需求已确认，待技术设计
- **产品名称**：PiWatch（暂定）
- **目标平台**：Raspberry Pi 4 Model B
- **技术栈**：Python
- **摄像头支持**：CSI 摄像头、USB 摄像头
- **存储方式**：本地 SD 卡
- **网络范围**：仅局域网

---

## 1. 产品概述

PiWatch 是一个运行在 Raspberry Pi 4 Model B 上的本地监控摄像头系统，提供实时预览、音视频录像、移动侦测、事件回放和异常邮件通知能力。

产品优先保证：

1. 部署简单；
2. 局域网内稳定可用；
3. 视频和音频默认保存在本地，不上传云端；
4. 支持 CSI 与 USB 两类摄像头；
5. 适合树莓派 4B 的性能和 SD 卡存储条件。

### 1.1 产品目标

- 用户在 30 分钟内完成树莓派监控服务部署。
- 用户可以使用手机或电脑浏览器查看实时画面。
- 用户可以手动录像，也可以在检测到移动或异常事件后自动录像。
- 用户可以在网页端查看、播放、下载和删除事件录像。
- 检测到异常事件后，通过 SMTP 发送邮件通知。
- 系统支持 7×24 小时运行，并在摄像头断开或服务异常时自动恢复。

### 1.2 非目标

- 第一阶段不做公网访问、端口映射或云端中转。
- 第一阶段不做专业级 NVR、多设备集中管理。
- 第一阶段不做默认开启的人脸识别、车牌识别等高风险识别能力。
- 不支持未经授权的偷拍、窃听或隐蔽采集。

## 5. 功能需求

### 5.1 首次安装与初始化

用户安装系统后，通过浏览器完成初始化，不需要手工编辑配置文件。

**流程**：

1. 树莓派启动 Python 服务。
2. 用户通过 IP 地址或 mDNS 地址打开初始化页面。
3. 创建管理员账号和密码。
4. 枚举并选择 CSI 或 USB 摄像头。
5. 枚举并选择 USB 麦克风，可选择暂不启用音频。
6. 配置分辨率、帧率、时区、录像目录和存储策略。
7. 配置 SMTP 邮件服务和收件人。
8. 预览视频和测试音频。
9. 保存配置并启动监控服务。

**验收标准**：

- 找不到摄像头或麦克风时，页面显示设备检查建议。
- 配置保存后重启服务仍然生效。
- SMTP 配置必须提供测试邮件功能。
- 初始化未完成时，服务不允许进入正式运行状态。

### 5.2 摄像头管理

系统需要统一抽象 CSI 和 USB 摄像头，使上层录像、预览和移动侦测逻辑不依赖具体设备类型。

**功能**：

- 枚举 CSI 摄像头和 V4L2 USB 摄像头。
- 显示设备名称、设备路径、支持的分辨率和帧率。
- 配置主摄像头。
- 检测摄像头断开并自动重连。
- 摄像头不可用时在网页和日志中显示明确原因。
- CSI 与 USB 摄像头在 MVP 中支持二选一；多摄像头同时运行放到 V2.0。

### 5.3 音频采集

系统支持 USB 麦克风采集声音，并将音频封装到录像文件中。

**功能**：

- 枚举 ALSA/PulseAudio 可用音频设备。
- 选择 USB 麦克风。
- 全局音频采集开关，默认关闭。
- 显示当前麦克风、采样率、音频状态。
- 实时预览支持单独静音，但不改变录像采集开关。
- 音频设备断开时，视频录像继续，并记录音频异常。
- 录像使用 AAC 或系统兼容的音频编码格式。
- 邮件通知默认不发送音频附件，只发送事件信息和可选缩略图。

**隐私要求**：

- 开启音频时需要二次确认。
- 页面持续显示“视频采集中”“音频采集中”等状态。
- 提供一键停止音视频采集。
- 用户需要根据部署地点确认录音、告知和授权要求。

### 5.4 实时预览

**功能**：

- 显示实时视频、摄像头名称、时间和音频采集状态。
- 支持视频静音和全屏。
- 支持截图。
- 支持开始/停止手动录像。
- 显示视频分辨率、帧率、连接状态和音频状态。

**建议技术方案**：

- CSI 摄像头：Python 调用 `libcamera`/`rpicam`。
- USB 摄像头：通过 V4L2 接入。
- USB 麦克风：通过 ALSA 接入。
- 视频编码：优先 H.264 硬件编码。
- 浏览器预览：MVP 使用 HLS；提供 MJPEG 作为兼容或低性能模式。
- 音视频录像：使用 FFmpeg 或 GStreamer 完成采集、编码、同步和封装。

**目标指标**：

- 局域网端到端延迟目标小于 2 秒。
- 720p 预览目标 15–30 FPS，实际取决于摄像头和编码参数。

### 5.5 录像策略

支持以下录像模式：

1. **手动录像**：用户点击开始，点击停止后生成录像文件。
2. **移动侦测录像**：检测到移动后自动开始录像。
3. **持续录像**：按固定时长切片，例如每 5 分钟一个文件；可在设置中开启。
4. **定时录像**：V1.1 支持按时间段录像。

每个录像文件保存以下信息：

- 摄像头 ID。
- 录像类型：手动、移动侦测、持续、定时。
- 开始时间和结束时间。
- 视频分辨率、帧率和音频状态。
- 文件大小和时长。
- 缩略图路径。
- 文件状态：正常、异常结束、待清理、已删除。

### 5.6 移动侦测与异常事件

MVP 使用帧差或背景建模实现轻量级移动侦测，避免在 Raspberry Pi 4B 上引入过重的 AI 推理负担。

**配置项**：

- 灵敏度：低、中、高。
- 最小变化面积。
- 触发持续时间。
- 事件结束等待时间。
- 事件冷却时间。
- MVP 默认检测全画面，V1.1 支持矩形区域。

**事件录像**：

- MVP 可先从事件发生时开始录制。
- V1.1 增加事件前缓存 0–10 秒和事件后录制 10–60 秒。
- 同一时间段的连续移动应合并为一个事件，避免生成大量碎片。

**异常事件类型**：

- 检测到移动。
- 摄像头断开。
- 音频设备断开。
- SD 卡空间不足。
- 录像服务异常退出。
- 设备温度过高。

### 5.7 事件中心

事件列表字段：

- 缩略图。
- 发生时间。
- 持续时长。
- 事件类型。
- 关联录像。
- 是否已查看。
- 是否保护，受保护事件不可被自动清理。

**操作**：

- 播放。
- 下载。
- 删除。
- 标记已查看。
- 保护/取消保护。
- 按日期和类型筛选。

### 5.8 SD 卡存储管理

MVP 只使用 SD 卡本地存储，不支持 USB 硬盘和 NAS 作为录像目录。

**配置项**：

- 录像目录。
- 最大占用比例，建议默认 70%。
- 最低剩余空间阈值。
- 普通录像保留天数。
- 是否允许删除未保护事件。

**自动清理规则**：

1. SD 卡使用率达到阈值时触发清理。
2. 优先删除最旧的普通录像。
3. 受保护事件不自动删除。
4. 清理后仍不足时暂停新录像并发送邮件告警。
5. 录像目录不可写时停止录像并在网页显示原因。
6. 启动时扫描录像目录，修复数据库与文件不一致的问题。

**SD 卡保护策略**：

- 录像按固定时长切片。
- 使用高耐久 SD 卡。
- 避免单个录像文件过大。
- 设置保留空间，防止系统因磁盘写满而异常。
- 提供存储健康状态和低空间告警。

### 5.9 邮件通知

异常事件发生后，通过用户配置的 SMTP 服务发送邮件。

**MVP 通知类型**：

- 移动侦测事件。
- 摄像头断开或恢复。
- 音频设备断开。
- SD 卡空间不足。
- 服务异常重启。
- 设备温度过高。

**配置项**：

- SMTP 主机。
- SMTP 端口。
- SSL/TLS 模式。
- 发件账号。
- 发件密码或应用专用密码。
- 收件地址。
- 邮件主题前缀。
- 通知开关。
- 同类事件冷却时间。
- 是否附带事件缩略图，默认开启。

**要求**：

- 邮件密码加密保存，不在页面回显。
- 提供发送测试邮件功能。
- 同类事件默认 5 分钟内合并，避免邮件轰炸。
- 邮件默认不附带音频文件。
- SMTP 失败时记录错误并在网页显示通知状态。

### 5.10 系统设置与安全

- 首次使用必须设置管理员密码。
- 密码使用安全哈希保存。
- 默认只监听局域网地址。
- 不提供公网端口映射和远程访问配置。
- 登录失败限流。
- 录像下载、删除、配置修改均需要鉴权。
- 录像文件不能被 Web 服务器直接列目录访问。
- 会话 Cookie 使用 HttpOnly 和 SameSite 属性。
- 日志不得记录密码、SMTP 密码、会话令牌或完整录像 URL。
- 设置页显示视频和音频采集状态。
- 提供一键停止音视频采集、重启服务和安全关机。

## 7. 技术方案建议

### 7.1 系统架构

```text
CSI 摄像头 / USB 摄像头        USB 麦克风（可选）
          │                         │
          └──── Python 设备适配层 ──┘
                 │
       ┌─────────┼─────────┐
       │         │         │
   实时流服务  移动侦测  录像服务
       │         │         │
       └─────────┴─────────┘
                 │
       SD 卡文件存储 + SQLite
                 │
      Web API / Web UI / SMTP 邮件
```

### 7.2 推荐模块

| 模块 | 职责 |
|---|---|
| Camera Manager | 枚举 CSI/USB 摄像头、设置参数、检测断连 |
| Audio Manager | 枚举 USB 麦克风、采集音频、管理音频状态 |
| Capture Pipeline | 统一输出视频帧、音频帧和时间戳 |
| Stream Service | 输出 HLS 或 MJPEG 实时流 |
| Recorder | 音视频编码、同步、切片和文件封装 |
| Motion Detector | 帧分析、事件开始/结束判定 |
| Storage Manager | SD 卡配额、索引、清理和一致性检查 |
| Event Service | 事件创建、合并、查询和保护 |
| Notification Service | SMTP 邮件、冷却和失败重试 |
| Web API | 鉴权、配置、录像和事件接口 |
| Web UI | 实时预览、事件回放和设置 |
| System Monitor | CPU、内存、温度、SD 卡和服务状态 |

### 7.3 运行环境

- Python 3.x。
- FastAPI 或 Flask 提供 Web API。
- SQLite 保存配置、事件和录像索引。
- FFmpeg 或 GStreamer 完成音视频编码、切片和封装。
- CSI 摄像头使用 libcamera/rpicam。
- USB 摄像头使用 V4L2。
- USB 麦克风使用 ALSA。
- 使用 systemd 管理服务，设置开机启动和失败重启。
- Raspberry Pi 4B 优先启用硬件 H.264 编码。

## 8. API 草案

统一前缀：`/api/v1`

| 方法 | 路径 | 说明 |
|---|---|---|
| GET | `/health` | 服务健康检查 |
| GET | `/status` | 获取设备、摄像头、音频、SD 卡和温度状态 |
| GET | `/cameras` | 获取可用 CSI/USB 摄像头 |
| PUT | `/camera` | 更新主摄像头配置 |
| GET | `/audio/devices` | 获取可用音频设备 |
| PUT | `/audio` | 更新音频开关和设备 |
| GET | `/stream` | 获取实时流信息 |
| POST | `/recordings/start` | 开始手动录像 |
| POST | `/recordings/stop` | 停止手动录像 |
| GET | `/recordings` | 分页查询录像 |
| GET | `/recordings/{id}` | 获取录像元数据 |
| GET | `/recordings/{id}/download` | 鉴权下载录像 |
| DELETE | `/recordings/{id}` | 删除录像 |
| GET | `/events` | 分页查询事件 |
| PATCH | `/events/{id}` | 标记查看或保护事件 |
| GET | `/settings` | 获取系统设置 |
| PUT | `/settings` | 更新系统设置 |
| POST | `/notifications/test-email` | 发送测试邮件 |

统一错误格式：

```json
{
  "error": {
    "code": "CAMERA_NOT_FOUND",
    "message": "未检测到摄像头设备"
  }
}
```

## 9. 非功能需求

### 9.1 性能

- 720p 实时预览在 Raspberry Pi 4 Model B 上稳定运行。
- CSI 摄像头 + USB 麦克风组合下，音视频录像保持可接受的同步误差。
- 移动侦测平均 CPU 占用目标低于 50%。
- 事件列表首屏加载目标小于 2 秒。
- 单次 API 查询最多返回 50 条记录，使用分页。

### 9.2 稳定性

- 摄像头断开后 60 秒内自动重连。
- 音频设备断开时视频录像继续，并产生告警。
- 录像进程异常退出后自动拉起。
- 断电重启后数据库不应损坏。
- 未完成录像标记为异常结束，不得阻塞后续录像。

### 9.3 资源与功耗

- 录像写入 SD 卡，按固定时长分片。
- 默认预留 SD 卡空间，避免磁盘写满。
- 使用高耐久 SD 卡，降低长期写入损耗。
- 设备温度过高时降低帧率或暂停非必要任务，并发送告警。

### 9.4 可维护性

- 关键模块提供单元测试。
- 日志分为 DEBUG、INFO、WARNING、ERROR。
- 提供诊断信息导出，但不得包含密码和隐私视频。
- 配置变更可追踪，支持恢复默认值。

## 12. 风险与应对

| 风险 | 影响 | 应对方案 |
|---|---|---|
| SD 卡长期写入损坏 | 数据丢失、系统不可启动 | 使用高耐久 SD 卡、录像分片、保留空间和存储健康告警 |
| Raspberry Pi 4B 性能不足 | 卡顿、丢帧、音视频不同步 | 720p 起步，启用硬件 H.264 编码，降低移动侦测采样率 |
| 光照变化导致误报 | 录像过多、邮件骚扰 | 灵敏度、最小面积、事件合并和冷却时间 |
| USB 麦克风兼容性问题 | 无音频或音频中断 | 启动时检测 ALSA 设备，设备断开时视频继续 |
| 网络波动 | 预览中断 | 自动重连，录像与预览解耦 |
| 设备过热 | 降频或崩溃 | 散热片/风扇、温度监测和降级策略 |
| 邮件服务配置错误 | 无法收到异常通知 | 提供测试邮件、失败重试和状态提示 |
| 公网暴露 | 隐私和安全风险 | 仅支持局域网，不提供端口映射功能 |
| 数据库与录像不一致 | 出现失效记录 | 启动扫描、定期校验和孤儿文件清理 |

## 14. 后续技术设计需确认

1. CSI 摄像头和 USB 摄像头在 MVP 中二选一，还是需要同时开启？当前 PRD 默认二选一。

## 16. YOLO 目标检测扩展

### 16.1 功能范围

- Web 端启用或关闭 YOLO 检测。
- 配置 YOLO 模型路径，树莓派 4B 默认建议使用 nano 模型。
- 配置目标类别，例如 `person`、`car`、`dog`。
- 配置置信度阈值和检测采样帧率。
- 使用归一化 ROI 矩形选择检测区域，坐标范围为 0 到 1。
- 配置报警冷却时间，避免同一目标连续发送邮件。
- 检测到匹配目标后创建 `yolo_target_detected` 事件。
- 当事件类型和 SMTP 均启用时发送邮件报警。

### 16.2 运行要求

- Python 依赖：`ultralytics`、`opencv-python-headless`。
- CSI 摄像头使用 `rpicam-vid` 输出 MJPEG 帧。
- USB 摄像头使用 OpenCV/V4L2 读取帧。
- YOLO 未安装时，系统仍可运行普通 Web 管理功能，并在状态页显示依赖不可用。
